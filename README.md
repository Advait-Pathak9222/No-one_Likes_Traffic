# ParkPulse Bengaluru

**Illegal Parking Impact Intelligence for Targeted Enforcement**  
Flipkart GRiD Round 2 · Theme 1 · Bengaluru Traffic Police

> **ParkPulse is not a heatmap.** It is a decision system for deciding which
> illegal-parking zones Bengaluru should solve first, at what time, with what
> action, and why.

---

## Product Snapshot

ParkPulse converts police violation records into an operational enforcement
plan:

| Layer | What it does | Output |
|---|---|---|
| Hotspot engine | Detects recurring illegal-parking pressure | 250 m cells, DBSCAN clusters |
| Bias correction | Reduces patrol/device visibility bias | exposure-adjusted ranking |
| Obstruction scoring | Estimates traffic-flow impact proxies | TORI, spillback, capacity loss |
| Forecasting | Predicts recurring pressure safely over time | Capture@K / NDCG@K validation |
| Action engine | Converts risk into police action | tow, patrol, fixed-window, field brief |
| Command center | Makes the result usable in a shift | map, queue, station cards, simulator |

**Core line:** a heatmap describes where violations were recorded; ParkPulse
prioritizes where limited enforcement creates the highest operational payoff.

---

## What Is Honest Here

The provided dataset contains enforcement records, not live traffic sensors.

ParkPulse therefore labels every claim:

- **Observed:** location, time, police station, violation text, vehicle class,
  validation status, repeat vehicle identifiers where available.
- **Modelled:** recurrence risk, hotspot persistence, exposure-adjusted priority,
  station/time-window demand.
- **Proxy:** obstruction risk, queue-spillback risk, equivalent lane recovery
  minutes, traffic-flow impact estimate.
- **Not claimed:** exact km/h improvement, exact queue reduction, exact delay
  reduction, or measured before/after enforcement impact.

That honesty is deliberate. The system is day-one deployable from available
records, and becomes measured once closure, speed, signal and workforce feeds
are integrated.

## Setup

Requirements:

- Python 3.10+
- Node.js 18+
- npm

Create the Python environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

LightGBM/XGBoost are optional. If native OpenMP is unavailable on macOS, the
forecasting code safely falls back to scikit-learn's
`HistGradientBoostingRegressor`.

## Validation Philosophy

Police teams do not need a perfect city-wide regression score before a shift.
They need the best first 10-20 actions.

ParkPulse therefore evaluates recurrence with time-safe top-K metrics:

- Capture@K
- NDCG@K
- Precision@K where applicable
- all-record vs high-confidence robustness checks

Raw density is treated as a strong baseline, not ignored. ParkPulse uses
recurrence prediction as the first layer, then adds obstruction severity,
exposure correction, confidence, resource constraints and action briefs.
